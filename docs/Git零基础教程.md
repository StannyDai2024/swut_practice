# Git 零基础入门教程

## 一、什么是 Git？

**Git 是一个版本控制工具**——你可以把它理解为一个"时光机"。

### 为什么要用它？

| 不用 Git | 使用 Git |
|---------|---------|
| 代码改丢了找不回来 | 随时回溯历史版本 |
| 多人修改同一文件容易冲突 | 每个人的工作清晰分开 |
| 备份靠复制粘贴 | 自动记录所有改动 |

> **简单说**：Git 就是一个帮你管理代码历史记录的工具。

---

## 二、核心概念

在学习命令之前，先了解三个重要区域：

```
工作区 → 暂存区 → 仓库区
```

- **工作区**：你正在编辑代码的地方（肉眼可见的文件夹）
- **暂存区**：准备保存的代码（类似"购物车"）
- **仓库区**：正式记录的历史版本（类似"订单记录"）

---

## 三、基本操作

### 1. 拉取代码（下载别人仓库的代码）

当你需要获取一个远程仓库的代码时：

```bash
git clone 仓库地址
```

**举个例子：**
```bash
git clone https://gitee.com/xxx/myproject.git
```

> 执行后，会在当前文件夹创建一个 `myproject` 文件夹，里面就是仓库的代码。

---

### 2. 创建自己的仓库

#### 方式一：把本地文件夹变成 Git 仓库

```bash
# 1. 进入你要管理的文件夹
cd 我的项目

# 2. 初始化仓库（只需执行一次）
git init
```

> 执行后，文件夹里会多一个 `.git` 隐藏文件夹，表示这个文件夹已被 Git 管理。

#### 方式二：从零创建全新的远程仓库

1. 在 Gitee/GitHub 网站上点击"新建仓库"
2. 填写仓库名称、描述
3. 创建后会得到一个仓库地址
4. 本地执行：
```bash
git clone 仓库地址
```

---

### 3. 提交内容（保存代码的改动）

提交代码有三个步骤：

```bash
# 步骤1：查看当前状态
git status

# 步骤2：将改动添加到暂存区
git add 文件名        # 添加单个文件
git add .            # 添加所有改动

# 步骤3：正式提交（创建快照）
git commit -m "提交说明"
```

**举个例子：**
```bash
git add index.html
git commit -m "完成了首页布局"
```

> **提交说明怎么写？** 用简短的话描述这次改了什么，越清晰越好。

---

## 四、常用命令速查表

| 命令 | 作用 |
|-----|------|
| `git clone 地址` | 下载仓库到本地 |
| `git init` | 初始化新仓库 |
| `git status` | 查看当前状态 |
| `git add .` | 添加所有改动到暂存区 |
| `git commit -m "说明"` | 提交改动 |
| `git push` | 将本地改动推送到远程 |
| `git pull` | 从远程拉取最新代码 |
| `git log` | 查看提交历史 |

---

## 五、常见场景

### 场景1：第一次push代码到新仓库

```bash
git remote add origin 仓库地址   # 关联远程仓库
git push -u origin main         # 推送到远程，第一次需要输入账密认证，后续不再需要
```

### 场景2：更新本地代码

```bash
git pull origin main
```

### 场景3：查看历史记录

```bash
git log
```

---

## 六、最佳实践

1. **每次提交只改一件事** —— 方便追溯问题
2. **提交说明要清晰** —— 写清楚"做了什么"而不是"改了啥"
3. **提交前先 `git status`** —— 确认要提交的内容
4. **每天开始工作前先 `git pull`** —— 保持代码最新

---

## 七、注意事项

- `.git` 文件夹不要手动修改或删除
- 提交说明不要写中文（很多公司习惯用英文）
- 敏感信息（密码、密钥）不要提交到仓库

---

## 八、学习建议

1. **动手实践**：看再多不如实际操作一遍
2. **从小项目开始**：用一个简单项目练习
3. **常用 `git status`**：养成习惯，每次操作前先看状态
4. **遇到问题先搜索**：Git 的问题大多数人都遇到过
